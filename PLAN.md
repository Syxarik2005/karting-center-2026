# Общий план реализации проекта «Волна» (Лекция 2)

## 1. Бэкенд (Go + PostgreSQL)
- **Фаза 1. Инфраструктура:** Инициализация Go-модуля, настройка логгера, конфигурации, генерация SQL-запросов (sqlc) и роутера.
- **Фаза 2. Авторизация:** Работа с JWT, логин по телефону, генерация OTP.
- **Фаза 3. Расписание (Read-Only):** Отдача списка слотов с фильтрами, пагинацией и джоином таблиц маршрутов и инструкторов.
- **Фаза 4. Бронирование:** Транзакционное создание брони (с проверкой лимитов досок и мест), логика отмены (ранняя/поздняя).

## 2. Клиентское мобильное приложение (Flutter)
- **Фаза 1. Инфраструктура:** Инициализация проекта, настройка слоев (Clean Architecture), роутинг, базовый UI-кит.
- **Фаза 2. Авторизация:** Экран SCR-001 (Вход/Регистрация), сохранение токена.
- **Фаза 3. Расписание:** SCR-002 (Список слотов), BS-001 (Фильтры), SCR-003 (Карточка слота).
- **Фаза 4. Бронирование:** SCR-004 (Оформление), SCR-005 (История броней), SCR-006 (Детали брони и отмена).
- **Фаза 5. Профиль:** SCR-007 (Настройки пользователя).
