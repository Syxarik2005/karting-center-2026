# 🏎️ Картинг-центр «Волна» — Разработка (Лекция 2)

Добро пожаловать в репозиторий бэкенда и клиента для картинг-центра **«Волна»**. Этот проект является практической работой в рамках **Летней Школы Surf 2026**.

В рамках данной ветки (`feature/client`) была проведена большая работа по исправлению критических багов интеграции, улучшению бизнес-логики и добавлению новых функций.

---

## 🛠️ Выполненные задачи (3 Бага + 3 Фичи)

Для каждого изменения подготовлен подробный отчет с описанием требований и шагов тестирования. Документы находятся в папке `docs/`:

### 🐛 Исправленные баги
1. **[Bug 1: Несовпадение полей БД и клиента](docs/bug1_db_fields_mismatch.md)**
   * **Проблема**: Мобильное приложение на Flutter не могло распарсить ответы от API из-за расхождений в ключах JSON.
   * **Решение**: Добавлен алиасинг в SQL-запросы (`max_seats`, `price_per_seat`, `duration_minutes` и т.д.) и корректный расчет `booked_seats`. Код репозитория обновлен через `sqlc`.
2. **[Bug 2: Бесконтрольная отмена бронирования](docs/bug2_cancel_window.md)**
   * **Проблема**: Отсутствовали ограничения на отмену броней (можно было отменить заезд за секунду до старта или даже в прошлом).
   * **Решение**: Добавлена строгая валидация: отмена брони невозможна, если до начала заезда осталось менее 2 часов.
3. **[Bug 3: Ошибка фильтрации по датам в GetSlots](docs/bug3_date_filter.md)**
   * **Проблема**: Если передавался `date_from` в будущем, но опускался `date_to`, бэкенд возвращал пустой список.
   * **Решение**: Сделан автоматический сдвиг `date_to` на 7 дней вперед относительно переданного `date_from`.

### ✨ Добавленные фичи
1. **[Feature 1: CORS Middleware](docs/feature1_cors.md)**
   * **Зачем**: Разрешены кросс-доменные запросы для удобного тестирования API прямо из браузера через локальные песочницы.
2. **[Feature 2: Система рейтингов и отзывов инструкторов](docs/feature2_rating.md)**
   * **Зачем**: Возможность оценить работу инструктора после заезда (оценка 1-5 + комментарий).
   * **Реализация**: Новая таблица `ratings`, триггер PostgreSQL для пересчета среднего рейтинга в таблице `instructors` и POST-эндпоинт `/bookings/{id}/rate`.
3. **[Feature 3: Эндпоинт профиля пользователя](docs/feature3_profile.md)**
   * **Зачем**: Получение актуальных данных клиента по токену авторизации.
   * **Реализация**: Добавлен GET-маршрут `/auth/profile`.

## 🚀 Технологический стек

* **Backend**: Go 1.26, Chi (Routing), pgx (PostgreSQL driver), sqlc (Type-safe SQL generation).
* **Database**: PostgreSQL 16 (с триггерами и хранимыми процедурами).
* **Containerization**: Docker & Docker Compose.
* **Testing Sandbox**: HTML5 / JavaScript (Vanilla CSS + Glassmorphism UI).

## 💻 Инструкция по запуску

### 1. Запуск бэкенда и БД через Docker
Убедитесь, что Docker запущен, и выполните команду в корне проекта:
```bash
docker compose up -d --build
```
* API бэкенда будет доступно по адресу: `http://localhost:8080`
* База данных автоматически инициализируется схемой и моковыми данными из `db_init.sql`.

### 2. Тестирование через интерактивную песочницу (Sandbox)
Для быстрого ручного тестирования разработан веб-интерфейс, который эмулирует мобильное приложение.
Вы можете открыть файл `app_sandbox.html` прямо в браузере или запустить локальный сервер:
```bash
python3 -m http.server 8081 --bind 0.0.0.0
```
Затем перейдите по адресу: [http://localhost:8081/app_sandbox.html](http://localhost:8081/app_sandbox.html)

**В песочнице можно:**
* Авторизоваться (тестовый OTP код: `0000`).
* Посмотреть актуальный профиль (Feature 3).
* Посмотреть список доступных слотов на карте и списком.
* Оформить бронирование и отменить его (с проверкой 2-х часового лимита).
* Оценить работу инструктора по бронированию (Feature 2).
