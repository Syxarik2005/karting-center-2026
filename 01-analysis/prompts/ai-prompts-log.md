# Промпты, использованные для генерации аналитических артефактов

> Данный документ фиксирует все ключевые промпты, отправленные ИИ-ассистенту
> при работе над аналитикой проекта «Картинг-центр Апекс».

---

## Промпт 1: Генерация User Stories

```
На основе брифа от заказчика Дениса (владельца картинг-центра «Апекс»)
и описания предметной области (domain-description.md), сгенерируй
набор User Stories для клиентского мобильного приложения.

Контекст:
- Приложение только для роли «Клиент» (R-028)
- Бэкенд — black-box (R-004)  
- Горизонт планирования — 7 дней по умолчанию (R-027)
- Отмена центром с push-уведомлением (R-008)
- Без онлайн-оплаты на MVP

Формат: "Как <роль>, я хочу <действие>, чтобы <цель>"
с критериями приёмки для каждой истории.
Приоритеты: Must / Should / Could.
```

---

## Промпт 2: Генерация основных сценариев

```
На основе User Stories и API-контракта (openapi.yaml для «Апекс»),
сгенерируй основные BDD-сценарии в формате Given/When/Then.

Обязательно покрыть:
1. Авторизацию (OTP)
2. Просмотр расписания с фильтрацией (7 дней, SHORT/LONG)
3. Успешное бронирование
4. Конфликт при бронировании (409)
5. Бронирование с прокатной экипировкой
6. Отмену клиентом (заблаговременная и поздняя)
7. Отмену центром по погоде (R-008)
8. Оценку маршала
9. Профиль и выход

Для каждого сценария указать альтернативные потоки.
```

## Промпт 3: Бриф на дизайн

```
Создай бриф на дизайн мобильного приложения для картинг-центра «Апекс».

Требования:
- Dark theme (уличный картинг, ночные заезды)
- Цвета: красный (гоночный), жёлтый (акцент), тёмно-синий (фон)
- Типографика: Inter (Google Fonts)
- Стиль: современный, динамичный, спортивный
- Целевая аудитория: 16-35 лет

Включить:
- Цветовую палитру с HEX
- Описание компонентов UI-кита (карточка слота, кнопка, статус-бейджи)
- Описание анимаций
- Референсы
```

## Промпт 4: ER-модель и Sequence Diagram

```
На основе брифа Дениса и уточнений скоупа (R-004, R-008, R-015, R-027, R-028),
создай:
1. ER-модель клиентской части (сущности от сервера + локальное состояние)
2. Sequence Diagram для процесса создания бронирования (createBooking)
   с тремя ветками: успех (201), конфликт (409), слот недоступен (410)

Формат: Mermaid для диаграмм.
```

## Промпт 5: OpenAPI-спецификация

```
Создай OpenAPI 3.0.3 спецификацию API для клиентского мобильного приложения
картинг-центра «Апекс».

Эндпоинты:
- GET /slots — список слотов с фильтрами (date_from, date_to, track_config)
- GET /slots/{slotId} — детали слота
- POST /bookings — создание бронирования (slot_id, gear_type)
- GET /client/bookings — бронирования клиента
- POST /client/bookings/{bookingId}/cancel — отмена брони
- POST /client/bookings/{bookingId}/rate — оценка маршала
- GET /client/profile — профиль клиента

Модели: Slot, Booking, Marshal, ClientProfile, TrackConfig, GearType, BookingStatus, SlotStatus
Ответы на ошибки: 409 (Conflict), 410 (Gone), 400 (Bad Request)
```
