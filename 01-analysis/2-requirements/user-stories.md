# User Stories — Клиентское приложение «Апекс»

> Приоритет: **Must** — обязательно для MVP, **Should** — желательно, **Could** — опционально.

---

## US-01 · Авторизация по номеру телефона (Must)

**Как** клиент картинг-центра,  
**я хочу** войти в приложение по номеру телефона (с подтверждением одноразовым кодом),  
**чтобы** мои бронирования были привязаны к моему аккаунту и я мог управлять ими.

**Критерии приёмки:**
- Клиент вводит номер телефона в формате +7 (XXX) XXX-XX-XX.
- Приложение отправляет запрос на генерацию OTP-кода.
- Клиент вводит 4-значный код, система проверяет его и возвращает JWT-токен.
- Токен сохраняется локально и используется для авторизованных запросов.
- При невалидном коде показывается ошибка, клиент может запросить повторную отправку.

## US-02 · Просмотр расписания заездов (Must)

**Как** клиент,  
**я хочу** видеть список доступных заездов на ближайшие 7 дней,  
**чтобы** выбрать удобное время и конфигурацию трассы.

**Критерии приёмки:**
- По умолчанию отображаются слоты на 7 дней вперёд от текущего момента (R-027).
- Каждая карточка слота содержит: время старта, конфигурация трассы (SHORT / LONG), имя маршала и его рейтинг, количество свободных картов (`available_karts` / `max_karts`).
- Если слотов нет — показан empty state «Пока нет доступных заездов».
- Слоты с `status = CANCELLED_BY_WEATHER` не отображаются в общем списке или помечены как недоступные.

## US-03 · Фильтрация слотов по дате и трассе (Must)

**Как** клиент,  
**я хочу** фильтровать заезды по дате и конфигурации трассы,  
**чтобы** быстро найти подходящий мне слот.

**Критерии приёмки:**
- Доступен фильтр по диапазону дат (расширение за пределы 7 дней).
- Доступен фильтр по конфигурации трассы: SHORT (новички) / LONG (опытные).
- При изменении фильтров список обновляется автоматически.
- Фильтры сбрасываются кнопкой «Сбросить».

## US-04 · Бронирование заезда (Must)

**Как** клиент,  
**я хочу** записаться на выбранный заезд, указав тип экипировки,  
**чтобы** забронировать себе место на трассе.

**Критерии приёмки:**
- Клиент видит детали слота: время, маршал, трасса, свободные карты, место сбора, стоимость проката.
- Клиент выбирает тип экипировки: «Своя» (OWN) или «Прокат» (RENTAL).
- Если прокатного фонда экипировки нет — опция «Прокат» недоступна с пояснением.
- При нажатии «Записаться» показывается индикатор загрузки.
- При успешном бронировании (201) — переход на экран деталей бронирования.
- При конфликте (409 — нет мест) — алерт «Места закончились», кнопка блокируется.
- При отмене слота (410 — слот недоступен) — алерт «Заезд был отменён», возврат к расписанию.

## US-05 · Просмотр истории бронирований (Must)

**Как** клиент,  
**я хочу** видеть список своих бронирований (предстоящие и прошедшие),  
**чтобы** отслеживать свои записи и управлять ими.

**Критерии приёмки:**
- Две вкладки: «Предстоящие» и «Прошедшие / Отменённые».
- Каждая карточка показывает: время заезда, трасса, маршал, статус (ACTIVE / CANCELLED_BY_CLIENT / CANCELLED_BY_CENTER / COMPLETED).
- Если заезд отменён центром — выделена причина отмены.
- Тап на карточку открывает детали бронирования.

---

## US-06 · Отмена бронирования клиентом (Must)

**Как** клиент,  
**я хочу** отменить свою запись заблаговременно,  
**чтобы** освободить место для других.

**Критерии приёмки:**
- Кнопка «Отменить» доступна только для бронирований со статусом ACTIVE.
- При нажатии — попап подтверждения с предупреждением о правилах отмены.
- Если до старта заезда менее 2 часов — отмена невозможна, показано сообщение «Слишком поздно для отмены» (бизнес-правило из бэкенда, ответ 400).
- При успешной отмене статус меняется на CANCELLED_BY_CLIENT, место освобождается.

---

## US-07 · Оценка маршала после заезда (Should)

**Как** клиент,  
**я хочу** оценить работу маршала после завершения заезда,  
**чтобы** помочь центру отслеживать качество инструкторов.

**Критерии приёмки:**
- Кнопка «Оценить маршала» доступна для бронирований со статусом COMPLETED, если оценка ещё не была поставлена.
- Шкала оценки: 1–5 звёзд.
- Текстовый комментарий — опциональное поле.
- Повторная оценка невозможна (кнопка скрыта после отправки).
- Средний рейтинг маршала обновляется на стороне бэкенда (триггер в PostgreSQL).

---

## US-08 · Просмотр профиля (Should)

**Как** клиент,  
**я хочу** видеть свои данные (имя, телефон, статус лояльности),  
**чтобы** убедиться в актуальности своего аккаунта.

**Критерии приёмки:**
- Отображаются: имя, телефон, статус «Постоянный клиент» (если `is_regular = true`).
- Возможность выйти из аккаунта (очистка JWT-токена).

---

## US-09 · Обработка отмены заезда центром / по погоде (Must)

**Как** клиент,  
**я хочу** быть уведомлён, если мой заезд отменён картинг-центром (например, из-за непогоды),  
**чтобы** не приехать зря.

**Критерии приёмки:**
- При отмене слота центром бронь получает статус `CANCELLED_BY_CENTER` с указанием причины (R-008).
- Клиент получает push-уведомление.
- Повторная запись на отменённый слот запрещена.
- В истории бронирований причина отмены выделена визуально.

---

## US-10 · Уведомления-напоминания (Could)

**Как** клиент,  
**я хочу** получать напоминание о предстоящем заезде (например, за 1 час),  
**чтобы** не забыть о записи и приехать вовремя.

**Критерии приёмки:**
- Push-уведомление за 1 час до старта заезда (конфигурируемый интервал).
- Уведомление содержит: время, место сбора, конфигурацию трассы.
