# Bug 1: Несовпадение полей БД и клиента

**Симптом/Цель:**
Мобильное приложение (клиент на Flutter) не могло распарсить ответы от сервера, так как названия полей в JSON отличались (например, `total_seats` вместо `max_seats`), а поле `booked_seats` (занятые места) вообще отсутствовало в ответе.

**Требования:**
Привести ключи JSON, возвращаемые бэкендом, в полное соответствие с моделью `SlotModel` из клиента на Flutter (`mobile_app/lib/data/models/slot_model.dart`).

**Реализовано:**
В `internal/repository/query.sql` изменены алиасы столбцов при выборке `ListSlots` и `GetSlotByID`:
- `total_seats AS max_seats`
- `(total_seats - free_seats) AS booked_seats`
- `price AS price_per_seat`
- `rental_price AS price_rental`
- `duration_min AS duration_minutes`
После чего выполнена генерация кода через `make sqlc`.
