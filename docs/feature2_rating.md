# Feature 2: Система рейтингов для инструкторов

**Симптом/Цель:**
В приложении отсутствовала возможность оценивать инструкторов после заезда (оставлять отзывы), что полезно для контроля качества.

**Требования:**
Реализовать эндпоинт для выставления оценки за прошедшее бронирование (оценка от 1 до 5 и опциональный комментарий).

**Реализовано:**
- В `db_init.sql` добавлена таблица `ratings` и триггер для автоматического пересчета среднего рейтинга инструктора (`rating` в таблице `instructors`).
- В `query.sql` добавлен запрос `CreateRating`.
- В `handler.go` добавлен обработчик `RateBooking` (эндпоинт `/bookings/{id}/rate`), принимающий рейтинг и проверяющий существование брони.

**Отправленные промпты:**
*(Пользователь)*: "написать 3 фичи"

**Ручная проверка:**
Выполнение POST запроса на `/client/bookings/{id}/rate` с телом `{"rating": 5, "comment": "Отлично!"}` добавляет запись и возвращает HTTP 201.
