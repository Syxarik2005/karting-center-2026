# Bug 2: Отмена бронирования в любое время

**Симптом/Цель:**
Пользователь мог отменить бронирование за секунду до начала заезда или даже для прошедшего заезда, так как отсутствовала валидация окна отмены.

**Требования:**
Запретить отмену бронирования, если до начала слота (start_at) осталось менее 2 часов.

**Реализовано:**
В обработчике `CancelBooking` (`internal/api/handler.go`) добавлена проверка разницы во времени:
```go
if time.Now().Add(2 * time.Hour).After(slot.StartAt.Time) {
    // Возвращается 400 Bad Request
}
```

**Отправленные промпты:**
*(Пользователь)*: "Исравить 3 разных бага"

**Ручная проверка:**
При попытке отменить ближайший слот, клиент получает ошибку с кодом `too_close`. Отмена слотов на завтрашний день проходит успешно.

Изменения будут закоммичены в общий пул по завершении.
