# Feature 3: Получение профиля пользователя

**Симптом/Цель:**
После авторизации клиент получает только токен и свои данные на момент входа, но у него нет возможности запросить актуальные данные своего профиля отдельно от логина.

**Требования:**
Реализовать GET эндпоинт `/auth/profile` для получения информации об авторизованном клиенте.

**Реализовано:**
В `handler.go` добавлен обработчик `GetProfile`, который извлекает `clientID` из JWT токена и запрашивает данные пользователя из базы данных через `GetClientByID`.

**Отправленные промпты:**
*(Пользователь)*: "написать 3 фичи"

**Ручная проверка:**
GET запрос на `/auth/profile` с валидным токеном возвращает данные пользователя в формате JSON. Запрос без токена возвращает 401 Unauthorized.
