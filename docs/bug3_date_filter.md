# Bug 3: Ошибка фильтрации слотов по датам

**Симптом/Цель:**
При передаче параметра `date_from` (но без `date_to`) в эндпоинт `/slots`, параметр `date_to` оставался по умолчанию `Now() + 7 days`. Если `date_from` был в будущем, то дата "до" оказывалась меньше даты "от", и база данных возвращала пустой массив.

**Требования:**
Корректно вычислять верхнюю границу `date_to`, если она не передана явно, относительно `date_from`.

**Реализовано:**
В `GetSlots` (`internal/api/handler.go`) при получении `date_from` обновляется и дефолтный `endDate` (`startDate + 7 дней`), если `date_to` не передано.

**Отправленные промпты:**
*(Пользователь)*: "Исравить 3 разных бага"

**Ручная проверка:**
Передача `?date_from=2026-08-01` возвращает слоты с этой даты на неделю вперед, вместо пустого массива.
