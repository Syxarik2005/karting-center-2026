# FEAT-002 — Бронирование заезда (SCR-002) + обработка конфликта 409

## Цель
Клиент выбирает экипировку (своя / прокат) и подтверждает запись. Ключевое —
честно обработать ситуацию, когда место заняли параллельно (гонка на
последний карт), а не притворяться, что бронирование всегда успешно.

## Требования
- `customer-brief.md` — «выбирал, со своей экипировкой он … или ему дать напрокат».
- `domain-description.md` — BR-01 (атомарность на бэкенде, клиент обязан
  корректно обработать отказ 409).
- `screen-registry.md` — SCR-002, состояния Content/Submitting/Error(Conflict)/Success.
- `openapi.yaml` — `POST /bookings`, ответы `201`, `409` (мест нет), `410` (слот исчез).

## Реализация
- `src/pages/SlotDetailsPage.tsx` — детали слота, radio-выбор экипировки,
  фиксированная кнопка снизу. `useMutation` из react-query для `createBooking`.
- Обработка ошибок **по коду**, не по факту ошибки вообще:
  - `NO_KARTS_AVAILABLE` → «Места закончились — кто-то забронировал раньше»
    + `invalidateQueries(['slot', slotId])` и `['slots']`, чтобы бейдж на
    экране сразу перерисовался (0 мест), а не остался врать.
  - `SLOT_GONE` (410) → «Заезд больше недоступен», кнопка блокируется.
- `src/api/mockApi.ts` → `createBooking` **реально** декрементирует
  `available_karts` в сид-данных при успехе — это значит, что если открыть тот
  же слот на другой вкладке после успешной брони, он честно покажет на 1
  место меньше (это не заглушка «всегда 201»).
- Успешная бронь → редирект на `/bookings/:id` с флагом `justBooked` для
  баннера подтверждения (закрывает состояние Success из ТЗ без отдельного модального экрана).

## Промпты (переданные ИИ-ассистенту)
```
1. "SCR-002 по ТЗ: экипировка своя/прокат radio-кнопками, кнопка "Записаться"
    внизу экрана фиксированная. Обработай 409 и 410 отдельно, с разными
    сообщениями, и обязательно инвалидируй кэш слота после конфликта — иначе
    бейдж на предыдущем экране будет врать про количество мест."

2. "Мок createBooking должен реально уменьшать available_karts, а не просто
    возвращать 201 — иначе тестирование конфликта 409 будет нечестным."
```

## Ручная проверка
1. Открыть слот `slot-3` (available_karts = 0 в сидах) напрямую по `/slots/slot-3`
   → кнопка должна быть задизейблена и подписана «Мест нет» без похода на сервер.
2. Открыть `slot-2` (1 место) → изменить в `mockData.ts` `available_karts` на 0
   вручную *после* открытия экрана (или дождаться параллельной брони в двух
   вкладках) → отправить форму → должно показать «Места закончились» текстом,
   а не белым экраном/консольной ошибкой.
3. Успешно забронировать `slot-1` → должен произойти редирект на
   `/bookings/:id` с зелёным баннером «Вы записаны!».
4. Вернуться на `/` → у `slot-1` должно быть на 1 место меньше, чем до брони.
