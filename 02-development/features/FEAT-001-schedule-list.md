# FEAT-001 — Расписание заездов (SCR-001)

## Цель
Клиент должен увидеть свободные слоты на ближайшие 7 дней, с возможностью
расширить период и отфильтровать по типу трассы, и понять по карточке —
стоит ли вообще тапать (мест нет / мало / заезд отменён погодой).

## Требования
- `01-analysis/0-customer-brief/customer-brief.md` — R-027 (горизонт 7 дней по умолчанию).
- `01-analysis/1-elicitation/domain-description.md` — BR-03.
- `01-analysis/4-design/screens/screen-registry.md` — SCR-001 (состояния Loading/Content/Empty/Error).
- `01-analysis/api/openapi.yaml` — `GET /slots` (`date_from`, `date_to`, `track_config`).

## Реализация
- `src/pages/ScheduleListPage.tsx` — список слотов, чипы периода (7/14/30 дней) и
  трассы (все/короткая/длинная), 4 состояния из ТЗ через react-query
  (`isLoading` → скелетон, `isError` → ErrorState с ретраем, пустой массив → EmptyState).
- `src/components/SlotCard.tsx` — карточка: время, трасса, маршал+рейтинг,
  бейдж доступности (зелёный / жёлтый при ≤2 местах / серый "мест нет"),
  отдельный бейдж для `CANCELLED_BY_WEATHER`. Карточка некликабельна, если мест
  нет или заезд отменён — чтобы не пускать на форму бронирования в тупик.
- `src/api/mockApi.ts` → `getSlots()` — фильтрует сид-данные по датам и трассе,
  дефолтный `date_to = now + 7d`, если не передан.

## Ручная проверка
1. `npm run dev` в `02-development/client`, открыть `/`.
2. Убедиться, что слот `slot-2` (1 место) — жёлтый бейдж, `slot-3` (0 мест) —
   серый и некликабельный, `slot-5` (погода) — отдельный янтарный бейдж и тоже
   некликабельный.
3. Переключить фильтр на "30 дней" → должен появиться `slot-8` (>7 дней от старта).
4. Переключить фильтр трассы на "Короткая" → в списке остаются только SHORT-слоты.
5. Временно подменить `getSlots` на `Promise.reject()` → должен показаться
   ErrorState с кнопкой "Повторить".
