# День 2 — Разработка · Картинг-центр «Апекс»

Клиентское React-приложение (мобильный форм-фактор), собранное поверх
результатов Дня 1 (`01-analysis/`) и OpenAPI-контракта в
`01-analysis/api/openapi.yaml`.

## Что внутри

| Что | Где |
| --- | --- |
| Архитектурный план | [`architecture/architecture-plan.md`](architecture/architecture-plan.md) |
| Схема данных (клиент) | [`architecture/data-model.md`](architecture/data-model.md) |
| Фичи (по одному .md на фичу, с промптами и ручной проверкой) | [`features/`](features/) |
| Код приложения | [`client/`](client/) |

## Реализованные фичи (4, при минимуме 3 из задания)

1. [FEAT-001](features/FEAT-001-schedule-list.md) — Расписание заездов (SCR-001)
2. [FEAT-002](features/FEAT-002-booking-flow.md) — Бронирование + обработка 409/410 (SCR-002)
3. [FEAT-003](features/FEAT-003-my-bookings-cancel.md) — Мои заезды + отмена (SCR-003/004)
4. [FEAT-004](features/FEAT-004-rate-marshal.md) — Оценка маршала (SCR-005)

Профиль (SCR-006) реализован как отдельный read-only экран без своего
FEAT-документа — там нет бизнес-логики или требований, которые стоило бы
отдельно фиксировать (просто вывод данных из `GET /client/profile`).

## Как запустить

```bash
cd 02-development/client
npm install
npm run dev       # http://localhost:5173
```

Бэкенда нет и не будет в рамках этой поставки (см. R-004) — `src/api/mockApi.ts`
имитирует контракт `openapi.yaml` в памяти браузера, включая коды ошибок
409/410/400, чтобы фичи выше можно было проверить руками без реального сервера.

```bash
npm run build      # прод-сборка, проверяет TS
npm run preview    # локальный прод-превью
```
